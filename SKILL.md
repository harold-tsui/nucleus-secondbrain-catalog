# SecondBrain Catalog · 组织规范与放置手册

> 这是一套**工具中立的目录规范**：定义一组"空货架"和"往上放东西的方法"。
> 任何 Agent 在任意电脑/服务器上整理第二大脑时，按本手册决策即可，无需重新设计。

---

## 0. 一句话心智模型

第二大脑 = **逻辑物流仓储**。`仓库(物理机) → 领域(L1) → 次级(L2) → 货架(L3) → [货位(L4)] → 包裹(项目/工作单元)`。
**治理层（L1–L4）= 编号货架，稳定；包裹内部 = 自治，不管。**

- **地图 ≠ 现场**：catalog 存文件的"家/归属/沉淀"；笔记/代码等**活数据(现场)**另有其地——笔记软件现场进 `D0-notes`，项目代码仓即包裹本身（包裹即检出，跨机靠 GitHub remote 同步）。
- **地图 ≠ 政府**：catalog 是疆域；Agent/运行时是在上面干活的班子，可换。

---

## 0.5 开工第一问：先确认仓库拓扑（建任何目录之前必做）

第二大脑 = **一套统一逻辑 catalog + 每台机器物化其子集**。设备不是目录层级，只是"检出了哪个子集"。所以在新机器上**动手前，先问客户**：

1. **有没有「中央库」？** = 完整副本/权威全集（数据中心、云盘全量、master vault）。在哪？路径/地址？
2. **有没有「兄弟库」？** = 其他设备各持子集的**同一个逻辑 catalog**。有哪些？各持哪些域/子集？
3. **本机该持哪个子集？** 按角色定（笔记本=热项目；服务器=已上线；公司机=公司域）。
4. **怎么同步？**（git / 加密盘 / rsync / Syncthing…；注意 PII 域走加密同步、不上公共 git）。

**据答案决定动作：**

| 情况 | 动作 |
|------|------|
| **有中央库** | **不从零建**。从中央库**克隆/同步**本机所需子集，**沿用其既有结构与编号**，不另起炉灶。 |
| **有兄弟库** | 登记进 `00-system/03-ops` 的设备/库注册表；协调子集边界，保证同一东西**跨库仍唯一归属**（不重复）。 |
| **都没有** | 本机即首个库（可担任中央库）。按 §2 空货架从零建，并在 `ops` 标注"我是 origin / 全集所在"。 |

> **为什么必须先问**：不问就建 = 制造孤岛/分叉/重复归属，直接违背唯一归属（P7）。宁可多问一句，不可建错根。

## 1. 结构铁律 R1–R5

- **R1** 治理层（L1–L4）一律 `NN-描述名`；**禁止裸项目 ID 当层名**。
- **R2** 项目/工作单元只作**包裹**，放在最深货架之下，内部不中央治理（包裹内自治：自己的 git 仓 / 功能层）。
- **R3** **L1/L2/L3 强制存在**；L4 可选（内容多才铺）。
- **R4** 仓库根是物理机器上的 vault 根目录，跨机各持子集，不计入逻辑编号。
- **R5** 命名：`NN-` 两位编号；年份固定 4 位 `2026`；**同一层只用一种排序轴**，但存量货架(编号)与流量货架(年份)可并存（年份自然排后）。`80-projects` 治理层一律 ASCII。中英文：英文为主，家庭/社交/健康/兴趣域内可中文。

## 2. 空货架（14 域骨架 · 直接照建）

> 实体空货架目录见 `blueprint/`，完整 L1-L3 去个人化说明见 `BLUEPRINT.md`。本节是 agent 放置时的速查版。

```
00-system/        系统区:01-inbox(唯一收件口)/02-governance(规范模板·治理仓)/03-ops(脚本·设备·凭证)
—— 生命/工作领域 ——
10-family/        家庭   本机·禁git   L3=成员/事项
20-social/        人际   L3=圈层(同学/工作/合作/朋友) + 人脉经营 + 社群
30-health/        健康   本机·禁git   L3=身体/心理(可 person 化)
40-self/          自我:01-values/02-goals/03-journal/04-个人信息(证件·本机敏感)
50-learning/      学习:01-active-learning/02-books/03-courses/04-skills
60-finance/       财务   本机·禁git:assets/[02 负债]/cashflow/[04 投资]/insurance/planning
70-career/        事业【行政口】:employment/owned-business/side-business-admin/business-network——**不含项目本体**
80-projects/      项目【项目口】:所有正式业务项目 SSOT;L2=归属·L3=年度+状态双轴(含 00-workbench)
90-lifestyle/     兴趣:运动/旅行/娱乐/饮食
—— 功能/元层 ——
A0-agents/        人格工作区:platforms(软件)/workspaces(各persona仓)/runtime(产物)
B0-knowledge/     跨域知识(被≥2域引用才上);研究型包裹按 1_raw→2_work→3_wiki→4_output 分层
C0-archive/       冷档案(保留原归属路径)
D0-notes/         笔记软件现场(现场≠地图):按软件分 obsidian/onenote/ima
```

**按需创建，不铺空壳**：货架用到才建；重要但稀疏的留空席位（如跳号 `02/04` 预留）。`00-` 永远留给该层的"共同体/通用/元"。
**PII 铁律**：`10-family / 30-health / 60-finance` 及 `40-self/04-个人信息`（证件）含隐私 → **只存本机，禁止提交 git**。

## 2.5 项目口 vs 行政口（核心二分）

- **80-projects = 项目口**：收**有明显项目特征**的——明确**输入/输出物** + 生命周期。所有正式立项的业务项目实体（SSOT）在此。
- **70-career = 行政口**：收**尚未成项目**的——行政/规划/日常/工作台 + 组织背景知识（雇主/客户/供应商）。
- **判据**：有明确输入/输出物 + 项目特征？→ `80-projects`；是日常/规划/工作台/背景知识？→ `70-career`。
- 二者**双向引用**：项目执行在 80，组织背景知识在 70，互链不重复（P7）。

## 3. 放置决策流程（拿到一个文件/项目，依次问）

> **首次在一台机器上动手前，先做 §0.5 开工第一问（确认中央库/兄弟库）。**

1. **是隐私吗？**（家庭/健康/财务/证件）→ 进对应本机域，禁 git。
2. **是软件活数据吗？**（vault/笔记本/DB）→ 进 `D0-notes` 现场，按软件分；不混进内容域。
3. **它当下主要被当作"什么"来检索？** → 定 **L1 域**（唯一归属的总判据）。
4. **是项目吗？**（有输入/输出物 + 生命周期）→ 走 §7：业务项目进 `80-projects`，可复用能力进 `00-system`，agent 进 `A0-agents`。
5. **否则**：域内属哪个稳定主体（某人/组织/职能）→ 定 **L2 席位**；同类怎么找回 → 定 **L3 货架轴**（§4）。
6. **跨域/边界模糊？** → 唯一归属（§5），一处放、他处留引用。

> 找不到合法位置时，进 `00-system/01-inbox` 暂存，事后分诊——**绝不新建 `misc/其他` 垃圾抽屉**。

## 4. 怎么选 L3 货架轴

**先判存量 / 流量：**
- **存量**（你*持有*的、靠"是什么"找）→ 按**类别**。例：资产、保单、档案、收藏。
- **流量**（*流过*你的、靠"哪一年"找）→ 按**年份**。例：账单、日记、体检、项目年份。

**5 种可选聚合轴**：① 时间(年份/历程) ② 类别(类型/学科) ③ 关系(圈层/对象/来源) ④ 职能(固定槽位) ⑤ 归属(原路径,归档用)。

**时间轴的切法**看"什么在随时间变"：纯流水/身份不变→年份；随时间变的是角色→历程（时间锚+角色名，旧阶段不可变）。

---

## 5. 七条通用规则（决策启发式）

1. **一等公民由分类定，不由热度定**：L2/L3 席位 = 稳定主体；热门不新增席位。
2. **聚焦靠"年份"自然实现，不靠维护看板**：项目按年度归档，当前年自然是热的、旧年自然下沉——零维护。热度**不改结构**。
3. **`00-` = 通用/兜底席位**：收"不属于任一主体"的；新类型加货架，不重构。
4. **唯一归属 + 让渡**：一物一家。**归位(move)**=放错了就移+记日志；**演进(evolve)**=孵化出新性质→原件留存、新域新建、双向链接，**不物理移动**（防断链）。
5. **重要但稀疏 → 保留席位 + 按需创建**：层级由重要性定，不由文件量定；重要域哪怕空着也占位（含跳号预留），不降级、不铺空壳。
6. **工具/现场 ≠ 地图**：外部 App（通讯录/相册/游戏库）、笔记软件持全量活数据；第二大脑只存**沉淀**，活数据进 `D0-notes`/外部 App，不镜像搬运。
7. **形式 ≠ 分类**：网页/PDF/视频等载体不单独建文件夹，进包裹的 `1_raw/` 按主题分；大文件走冷热分层（只存索引卡+外链）。

## 6. 如何判断一个文件夹的"地位"

地位 = **价值定级**，由位置决定，供 Agent 判断该投入多少、是否该浅出/归档：

- **越浅越重要**：顶层 = 高价值高频；越深 = 越低频/越接近档案。**深度 = 访问代价 = 价值**——所以热项目用年份让当前年浅出，不埋深。
- **席位 vs 包裹**：L2/L3 是"一等公民席位"（稳定、有地位）；包裹（具体项目）地位随其所在货架与生命周期浮动。
- **地位向下传导**：父节点地位决定整棵子树的地位下限；提升父节点 = 提升子树。
- **热度 ≠ 地位**：再忙的项目也不因热度获得更高结构地位，只靠"当前年货架"自然浮顶。
- **判断某项目文件夹地位时**：看①它在哪个域（域的重要性）②它在第几层（越浅越重）③它是活跃/沉淀/封存哪个温度。

## 7. 各类项目（包裹）落哪

**判据先行**：有明确输入/输出物 + 生命周期 → 项目口；否则行政/规划/工作台 → 留 `70-career`。

| 项目性质 | 落点 |
|---------|------|
| 业务项目（自有事业/雇主/副业/个人发展，正式立项） | `80-projects/{归属}/{年度}/包裹` |
| 业务项目（未立项：孵化/售前/预研） | `80-projects/{归属}/{状态}/包裹`（年度+状态双轴并存） |
| 可复用**治理能力**（规范/框架/skill） | `00-system/02-governance/{独立仓}`（如 nucleus 家族） |
| 可复用**工具**（脚本/工具集） | `00-system/03-ops/tools/{独立仓}` |
| 研究/学习专题 | `50-learning/01-active-learning/{专题}/`（功能层包裹） |
| Agent/系统项目 | `A0-agents/workspaces/{persona}/` |
| 健康/财务/家庭计划 | 对应生命领域内 |
| 完成归档 | `C0-archive/projects/{原归属}/{项目}-{年}` |

> **判据只问"它是什么"，不问"谁用它"**：只服务一个归属 → 80-projects/{归属}；被多归属/agent 复用 → 00-system。

## 8. 放置前自检清单

- [ ] 这东西**唯一**的家是这里吗？别处有没有重复？
- [ ] 隐私域/证件没被推上 git？软件活数据进了 `D0-notes` 而非内容域？
- [ ] 治理层都是 `NN-名`？项目 ID 只出现在包裹层？`80-projects` 治理层全 ASCII？
- [ ] L1/L2/L3 三层齐了？没硬铺空 L4？
- [ ] 选的 L3 轴符合"它怎么被找回"？项目用年度让当前年浅出？
- [ ] 项目分清了项目口(80)/行政口(70)？跨域的他处留引用了吗？
- [ ] 没有制造 `misc/其他` 抽屉？
